Replit

查帳號設定與權限: https://replit.com/account

在 Replit 平台上學習 Python: https://replit.com/learn/100-days-of-python

本課程共有兩種方式利用 Replit 與 Github 建立並維護網站內容, 主要的網站內容 (靜態) 都以 Github Pages 為主, 而在 Replit 所啟動的動態網站則充作網站內容編輯之用:

第一種方法, 在 Replit 中建立 git 倉儲與網站所需內容, 再透過 git remote add 與 git push -u 與 Github 中的空倉儲整合, 其中的 cmsimde 可以直接作為倉儲目錄或者作為倉儲的子模組:

登入 Replit 後, 設法建立一個倉儲與遠端 Github 上的空倉儲同步.

為確保使用 "main" 作為主分支名稱，可以在 "git init" 之後加上 "git branch -m main" 指令。這將會將預設的分支名稱從 "master" 改為 "main"。操作流程如下：

1. 在 Replit 的終端機（Terminal）中，確定您目前所在的目錄是您想要作為 Git 倉儲的目錄。如果不是，您可以使用 "cd" 指令切換到正確的目錄。

2. 初始化 Git 倉儲：
- 執行指令 "git init"：這會在目前的目錄下初始化一個新的 Git 倉儲。

3. 將預設分支名稱改為 "main"：
- 執行指令 "git branch -m main"：這將把預設分支從 "master" 改為 "main"。

4. 連結到 GitHub 倉儲：
- 在 GitHub 上創建一個空的倉儲（Repository），並取得該倉儲的 URL。
- 在終端機中，執行指令 "git remote add origin <GitHub倉儲URL>"：這會將您的本地倉儲與 GitHub 倉儲連結起來。

5. 設定提交身分：
- 在 Replit 的右側窗格中，點擊「Secrets」選項卡。
- 在「Environment variables」區域，設定一個名為 "GIT_COMMITTER_NAME" 的變數，並設定您的名字。
- 設定一個名為 "GIT_COMMITTER_EMAIL" 的變數，並設定您的郵件地址。

6. 將本地的更改提交到 GitHub：
- 執行指令 "git add ."：將所有更改添加到暫存區。
- 執行指令 "git commit -m "Initial commit""：將更改提交到本地儲存庫。
- 執行指令 "git push -u origin main"：將本地的更改推送到 GitHub 倉儲。

現在，您的本地 Git 倉儲和 GitHub 上的倉儲已經連結起來，並且已設定提交身分，同時使用 "main" 作為主分支名稱。您可以在 Replit 上進行更改、提交和推送，並保持與 GitHub 上的倉儲同步。

第二種方法則是在 Github 先建立好網站倉儲, 可以將 cmsimde 作為倉儲目錄或者作為子模組, 並利用 Github Pages 伺服倉儲中的靜態網站 (也就是 content 目錄中的超文件), 之後再將倉儲 import 進入 Replit 執行倉儲中的動態網站進行內容的編輯與改版.

注意: 利用 Replit 維護 Github 倉儲時, git add 與 git commit 必須透過 Tools 中的 Git 頁面完成, 然後 git push 則可以在 Git 頁面或 Shell 頁面執行 git push.
